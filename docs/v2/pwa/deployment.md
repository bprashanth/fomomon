# PWA Deployment — Netlify (static pre-built)

The Flutter web app is built locally and the compiled output is committed to the
repo. Netlify serves it directly with no build step.

---

## What is deployed

Only `fomomon/build/web/` is served — the compiled, minified Flutter web bundle.
The `admin/` directory and all other repo contents are never touched by Netlify.

---

## One-time setup

### 1. gitignore

`fomomon/.gitignore` has `/build/` to exclude native build artifacts (Android,
iOS). The web output is force-added once to make git track it:

```bash
git add -f fomomon/build/web/
```

After the initial commit, git tracks those files and gitignore no longer applies
to them. Subsequent builds just show the diffs normally.

### 2. Netlify config (`netlify.toml` at repo root)

```toml
[build]
  base    = "fomomon"
  publish = "build/web"
```

`base = "fomomon"` scopes Netlify's working directory to the Flutter app subfolder,
preventing it from seeing `requirements.txt` (the admin backend's Python deps) at the
repo root and trying to install them. `publish` is relative to `base`.

No build command — Netlify serves the pre-built output as-is.

### 3. SPA routing (`fomomon/web/_redirects`)

```
/*    /index.html   200
```

Flutter web is a single-page app. This file tells Netlify to serve `index.html`
for any path, so deep links and refreshes work. Flutter copies it into
`build/web/` automatically during `flutter build web`.

### 4. Connect repo in Netlify

- **New site → Import from Git → select repo**
- **Branch**: `main`
- **Build command**: leave blank
- **Publish directory**: leave blank (read from `netlify.toml`)
- Click **Deploy**

Netlify auto-provisions HTTPS (required for GPS, camera, service worker, and
PWA install prompt — see `testing.md`).

## Deploying an update

```bash
cd fomomon/fomomon
flutter build web --release
cd ../..
git add fomomon/build/web/
git commit -m "Rebuild web"
git push origin pwa
```

Netlify picks up the push automatically and redeploys within ~30 seconds.

---

## PWA installability

A `--release` build satisfies all three Chrome PWA install requirements:

| Requirement | How satisfied |
|---|---|
| HTTPS | Netlify auto-provisions TLS |
| `manifest.json` with `display: standalone` + 192/512 icons | Present in `web/manifest.json` |
| Service worker with fetch handler | `flutter_service_worker.js` generated by `--release` build |

On Android Chrome: three-dot menu → **Install app** → app opens without browser bar.

> **Note**: The service worker is only present in `--release` builds. `flutter run -d chrome`
> (debug mode) does not generate it, so the install prompt will not appear during local dev.

## What the build output contains

```
build/web/
  index.html                  ← entry point; served for all routes
  main.dart.js                ← compiled Dart app (dart2js, minified)
  flutter.js                  ← Flutter web bootstrap
  flutter_bootstrap.js
  flutter_service_worker.js   ← PWA service worker (Workbox-based)
  manifest.json               ← PWA manifest (display: standalone)
  _redirects                  ← Netlify SPA routing rule
  favicon.png
  icons/                      ← Icon-192/512 + maskable variants
  assets/                     ← fonts, images, asset bundle
  canvaskit/                  ← CanvasKit WASM renderer
```
